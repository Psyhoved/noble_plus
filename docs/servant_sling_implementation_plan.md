# План реализации: праща как основное оружие слуги

Подготовлено для передачи другому разработчику.

## Контекст задачи

Цель: изменить стартовый отряд происхождения **«Дворянин +»** так, чтобы у персонажа-слуги (`bros[4]`) праща была **основным** оружием, а не кинжал.

Текущая реализация:
- слуга создаётся в `onSpawnAssets()` сценария `noble_plus_scenario`;
- сейчас слуге явно экипируется кинжал через `items4.equip(this.new("scripts/items/weapons/dagger.nut"));`.

## Где вносить изменения

Основной файл:
- `mod/mod_noble_plus/scripts/scenarios/world/noble_plus_scenario.nut`

Сопутствующие обновления документации:
- `README.md` (краткое описание фичи в корневом readme)
- `docs/README.md` (описание происхождения и перечень отличий)
- при необходимости `docs/ROADMAP.md` и/или `docs/JOURNAL.md` (зафиксировать изменение после внедрения)

> Важно: в проекте закреплён паттерн стабильности сценария (см. `docs/TECH.md`) — не трогать `create()/onInit()/onSpawnPlayer()` без необходимости, задача локальна для экипировки слуги.

## Пошаговый план

### Шаг 1. Проверить корректный script id пращи в игре/Legends

Перед правкой кода подтвердить, какой item-скрипт использовать для пращи:
- для текущей сборки подтверждён `scripts/items/weapons/legend_northern_sling.nut`;
- fallback-кандидаты в архивах Legends: `legend_sling.nut`, `legend_slingshot.nut`.

Критерий готовности:
- найден валидный путь к предмету пращи, который не вызывает ошибку `Cannot find script`.

### Шаг 2. Заменить экипировку слуги: кинжал → праща

В блоке `bros[4] — Слуга`:
1. Удалить/заменить строку экипировки кинжала.
2. Экипировать пращу как оружие в руку (`equip`), чтобы она попадала в активный слот и считалась основным оружием.
3. Оставить остальной стартовый набор без изменений (броня, шляпа, еда, traits, perk, позиция построения), если нет отдельного дизайна на перебаланс.

Рекомендуемый вид изменения:
- было: `items4.equip(this.new("scripts/items/weapons/dagger.nut"));`
- стало: `items4.equip(this.new("scripts/items/weapons/legend_northern_sling.nut"));`

### Шаг 3. Проверить боезапас/совместимость пращи

Праща может требовать боеприпасы в зависимости от реализации в Legends.

Проверить после старта кампании:
- оружие действительно активно у слуги;
- доступны атакующие умения пращи;
- нет проблем с отсутствием боеприпасов на первом бою.

Если боеприпасы обязательны и не выдаются автоматически:
- добавить стартовые боеприпасы слуге или в stash (минимально необходимый объём),
- задокументировать это как часть изменения.

### Шаг 4. Обновить текстовые описания фичи

Синхронизировать пользовательские тексты, где сейчас написано про кинжал:
- описание происхождения в `m.Description` внутри `noble_plus_scenario.nut`;
- `README.md` (раздел «Особенности»);
- `docs/README.md` (раздел с перечислением различий относительно ванили).

Формулировка:
- заменить «слуга вооружён кинжалом» на «слуга вооружён пращой».

### Шаг 5. Регрессионная проверка сценария

Минимальный smoke-test:
1. Игра загружается без `ERROR` уровня `SQ`.
2. Происхождение «Дворянин +» отображается корректно.
3. Новая кампания стартует без зависаний.
4. У слуги в тактическом бою экипирована праща, атака доступна.
5. Условие game-over при смерти дворянина не нарушено (быстрый sanity check).

### Шаг 6. Зафиксировать результат в документации разработки

После успешного теста внести запись:
- `docs/JOURNAL.md` — что изменено, где, как проверено;
- при использовании трекера этапов — отметить релевантный пункт в `docs/ROADMAP.md`.

## Риски и нюансы

1. **Неверный путь к скрипту пращи**
   - Симптом: ошибка загрузки/экипировки предмета.
   - Решение: подтвердить path по фактическим скриптам Legends.

2. **Праща без боезапаса на старте**
   - Симптом: оружие в слоте есть, но атаковать нельзя.
   - Решение: добавить стартовый боезапас.

3. **Рассинхронизация кода и текстов**
   - Симптом: в игре праща, а в описании всё ещё кинжал.
   - Решение: обязательный проход по списку файлов из Шага 4.

---

## Критерий приёмки задачи

Задача считается выполненной, когда одновременно соблюдены все условия:
- в коде слуга стартует с пращой в активном слоте оружия;
- кампания стабильно стартует без новых ошибок сценария;
- в бою праща функциональна (атака доступна);
- описание происхождения и проектная документация обновлены и не упоминают кинжал как стартовое оружие слуги.
